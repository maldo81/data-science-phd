---
description: Generate or refresh a project PRD
argument-hint: [output-path]
---

Create a PRD using project context and current requirements.

Default output path:
- `.project/specs/prd.md`

If `$ARGUMENTS` is provided, use that path instead.

Ensure the PRD includes:
- summary
- problem statement
- scope and non-goals
- requirements
- risks and mitigations
- validation plan
